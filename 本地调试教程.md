# Vibe Kanban Web Companion 本地调试教程

本教程将指导您如何在本地环境中调试 Vibe Kanban Web Companion 项目。

## 环境要求

- Node.js (推荐 v16+)
- pnpm (包管理器)
- Git

## 快速开始

### 1. 克隆项目

```bash
git clone https://github.com/SatanHJ/vibe-kanban-web-companion-vue.git
cd vibe-kanban-web-companion-vue
```

### 2. 安装依赖

```bash
pnpm install
```

### 3. 启动开发服务器

项目使用 Turbo 构建系统，支持并行开发多个应用。

```bash
# 启动所有应用的开发服务器
pnpm run dev
```

### 4. 构建项目

```bash
# 构建所有组件
pnpm run build
```

## 项目结构

```
.
├── apps/                    # 示例应用
│   ├── cra/                # Create React App 示例
│   ├── next/               # Next.js 示例
│   └── remix/              # Remix 示例
├── packages/               # 核心包
│   └── vibe-kanban-web-companion/  # 主包源码
└── parent-example.html     # 父窗口示例
```

## 调试指南

### 调试核心包

1. 进入核心包目录：
```bash
cd packages/vibe-kanban-web-companion
```

2. 查看包的 README 了解具体用法

### 调试示例应用

#### Create React App 示例

```bash
cd apps/cra
pnpm run dev
```

#### Next.js 示例

```bash
cd apps/next
pnpm run dev
```

#### Remix 示例

```bash
cd apps/remix
pnpm run dev
```

### 使用父窗口示例

打开 `parent-example.html` 文件在浏览器中，配合运行的示例应用进行调试。

## 常用命令

| 命令 | 说明 |
|------|------|
| `pnpm run dev` | 并行启动所有应用的开发服务器 |
| `pnpm run build` | 构建所有组件 |
| `pnpm run lint` | 运行 ESLint 检查 |
| `pnpm run release` | 发布新版本 |

## VSCode 调试配置

项目包含 `.vscode` 目录，可以直接使用 VSCode 的调试功能。

1. 打开项目
2. 按 F5 或点击调试面板
3. 选择相应的调试配置

## 集成到您的项目

在您的 React 应用中添加 Vibe Kanban Web Companion：

```bash
npm i vibe-kanban-web-companion
```

然后在入口文件中引入：

```jsx
import { VibeKanbanWebCompanion } from 'vibe-kanban-web-companion';

// 在根组件中添加
<VibeKanbanWebCompanion />
```

详细集成说明请参考 [packages/vibe-kanban-web-companion/README.md](packages/vibe-kanban-web-companion/README.md)

## 常见问题

### 依赖安装失败

确保使用 pnpm 而不是 npm 或 yarn：
```bash
npm install -g pnpm
pnpm install
```

### 端口冲突

如果默认端口被占用，请检查各个应用的配置文件并修改端口。

### 热重载不工作

1. 检查是否使用了正确的开发命令
2. 确保没有其他进程占用文件监听
3. 尝试删除 `node_modules` 和锁文件后重新安装
